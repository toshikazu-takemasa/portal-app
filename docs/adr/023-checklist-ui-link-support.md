# ADR-023: チェックリストの操作性向上（リンク対応とクリック領域の限定）

日付: 2026-04-23  
状態: 提案

## 背景

現在のチェックリスト UI では、行全体がクリックイベントに反応してチェックの切り替え（トグル）を行う仕様となっている。これにより、以下の不便が生じている。

1. **誤操作**: 行のどこを触ってもチェックが変わってしまうため、スクロール時などに誤ってチェックを付けてしまう。
2. **リンクの困難さ**: タスク名に URL を含めても、クリックが行全体のトグルに奪われてしまい、リンクを開くことが難しい。

## 決定

**チェック操作の反応領域をチェックボックスのみに限定し、タスク名でのリンク表示をサポートする。**

### 1. クリック領域の限定
- `<li>` 要素に設定されていた `onClick` イベントを、チェックボックス（アイコン部分）のみに移動する。
- 行の他の部分は、将来的にテキスト選択やリンククリックができるよう解放する。

### 2. リンク表示のサポート
- タスク名（`item.title`）に含まれる URL を自動的に検出し、クリック可能なリンクとしてレンダリングする。
- あるいは、リンクが存在する場合に「リンクボタン（外部アイコン）」を表示する。

## 修正内容

- `pages/checklist.tsx` のレンダリングロジックを修正。
- `item.title` をパースして、Markdown ライクなリンクまたは生の URL を `<a>` タグに変換するヘルパーを導入する。

## トレードオフ

- **操作の厳密性**: 以前は「どこでも押せばチェックできる」という手軽さがあったが、今後は「チェックボックスを正確に押す」必要がある。これはモバイルデバイス等での操作性に影響する可能性があるが、誤操作防止と機能拡張のために優先する。

## 将来の再評価タイミング

- モバイルでの操作が極端に困難になった場合、チェックボックスのヒット領域（Padding）を広げるなどの微調整を行う。
