# ADR-026: AIチャット会話内容のリモート保存

日付: 2026-05-01
状態: 提案

## 背景

ADR-016 で定義されたAIチャット機能の実装アーキテクチャでは、データストレージとして「一時的には localStorage または IndexedDB で保持する」「永続化が必要な場合は Vault 配下に書き込む」とされていた。
しかし、GitHub Issue #2 にて「aiチャットの会話内容をリモートに保存」することが求められている。
ブラウザのローカルストレージのみに依存していると、別端末からのアクセス時に会話の文脈（履歴）が引き継げないため、リモートリポジトリ（GitHub等）への永続化・同期の仕組みが必要である。

## 決定

**AIチャットの会話履歴（History）をJSON形式でVault（GitHubリポジトリ等）に保存し、アプリ起動時・チャット画面マウント時にリモートから取得（フェッチ）する仕組みを構築する。**

### アーキテクチャ

1. **保存形式**
   - パス例: `vault/chat/history.json`
   - データ構造例: `[{ role: 'user', content: '...' }, { role: 'assistant', content: '...' }]` の配列、あるいは会話スレッド（セッション）ごとに分割された構造。
2. **同期タイミング**
   - **保存時**: AIの応答が完了（ストリーミング終了）したタイミング、またはユーザーが明示的に「保存」ボタンを押したタイミングで、GitHub API等を経由してリモートリポジトリへコミット（Upsert）する。
   - **読み込み時**: チャット画面を開いた際、リモートストレージから最新の `history.json` を取得し、ステートに復元する。

## トレードオフ

- **メリット:** 複数端末間でのチャット履歴の共有・継続が可能になる。GitHubをバックエンドとすることで新たなDBを契約するコストがかからない。
- **デメリット:** 会話が長引くたびにリモート保存APIを叩くと、GitHub APIのレートリミットやI/Oコストが増加する。
  - *対策案:* 定期的なオートセーブ（デバウンス処理）や、一定のメッセージ数ごとのバッチ保存を検討する。

## 次のステップ

- `pages/api/chat/history.ts` などの履歴保存用プロキシAPIの実装。
- チャットUIコンポーネントにおける、マウント時の履歴読み込みと、メッセージ追加時の自動保存ロジックの実装。
