# ADR-015: チェックリストのVault同期仕様の定義

日付: 2026-04-16
状態: 採用

## 背景

現在、チェックリスト（Daily Tasks等）は以下の2箇所にデータが存在します。
1. **Vault (PortalConfig)**: `dailyTasks` 配列など。いわゆる「テンプレート（マスター）」としての役割。
2. **localStorage**: 今日のタスク一覧および完了状態（`completed: boolean`）のキャッシュ。

既存の実装では、Vaultのテンプレートと `localStorage` キャッシュをマージする際に、「新しく追加されたID」は反映されるものの、「Vault側から削除・名称変更された古いID」が `localStorage` 内に残り続ける仕組みになっていました。これにより、削除したはずのチェックリストが表示され続けたり、同名のタスクが重複登録されたように見える不具合が発生しています（「 Vault を更新した場合もサイトキャッシュがあるのか重複登録のような形になる」問題）。

## 決定

**Vault（PortalConfig側）の定義を「100% 正（master）」として振る舞うようにマージロジックを変更する。**

### 具体的な変更内容
- `localStorage` に保持している当日のチェックリストの完了ステータスキャッシュから、**現在Vaultに存在しているIDのタスクのみ**を抽出してマージします。
- Vaultテンプレートに存在しない古いID・キャッシュは無視（破棄）し、画面に表示しないようにします。
- これにより、ユーザーがVaultを更新した瞬間に、古いタスクは消え、新しいタスクが未完了状態で自然に反映されるようになります。

## トレードオフ

- **メリット:** Vaultリポジトリでの変更が、画面上に素直に反映されるため、ユーザーの期待に沿った動作になります。重複バグも解消されます。
- **デメリット:** Vault側の操作ミスで意図せずタスクを削除し、直後に復元した場合などでも、別のIDになっていればキャッシュ（完了状態）は失われます。しかし基本的にチェックリストは1日単位の短命データであるため、致命的ではありません。
