# ADR-016: AIチャット機能の実装アーキテクチャ

日付: 2026-04-16
状態: 採用

## 背景

現在、Portalには設定ファイル(Vault)で指定したAPIキーとペルソナを利用して日記を自動要約する機能（`/api/summarize`）が存在します。しかし、AIとの双方向の対話を行う「AIチャット」機能が実装されておらず、単発の要約処理にとどまっています。
ユーザーからの「AIチャット機能が未実装なので開発を進めたい」との要望を受け、対話機能のアーキテクチャを決定する必要があります。

## 決定

**AIチャット専用の画面とストリーミング対応のAPIルートを構築し、対話履歴をプロファイル単位で永続化する。**

### コンポーネント設計
1. **フロントエンド (`pages/chat.tsx`)**: 
   - 対話履歴の表示とメッセージ入力フォームを提供するUI。
   - `Vercel AI SDK` 等を利用するか、標準の `fetch` を使ったストリーミングパーサーを実装し、タイピングアニメーションを伴うレスポンスを実現する。
2. **バックエンド (`pages/api/chat.ts`)**:
   - `Profile` 内の `ai_persona`（APIキーやモデル設定）を利用して、LLM（Anthropic 等）へリクエストをプロキシするエンドポイント。
3. **データストレージ**:
   - 対話の履歴は一時的には `localStorage` または IndexedDB で保持する。
   - 永続化が必要な場合は Vault 配下（例: `vault/chat/history.json`）に書き込む仕組みを提供する（VaultのI/Oコストとの兼ね合いで頻度はバッチ化するなど調整）。

## トレードオフ

- **メリット:** 既存の `ai_persona` 設定をそのまま使い回せるため、ユーザーは追加設定なしでチャット機能を利用できる。
- **デメリット:** Vercel プラットフォーム以外（特に現状のCloudflare環境）でデプロイする場合、ストリーミングの動作保証が必要（Edge Runtimeへの対応等）になる場合がある。
