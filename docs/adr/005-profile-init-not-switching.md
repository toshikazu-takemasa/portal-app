# ADR-005: プロファイルはUIで切り替えず初期設定で決定する

日付: 2026-04-07  
状態: 採用

## 背景

Phase 1〜2 ではヘッダーに仕事/個人のプロファイル切り替えボタンを実装していた。
しかし実際の利用シーンを検討すると、同一セッション中に仕事とプライベートを頻繁に切り替えることは想定されない。

## 検討した選択肢

1. **UIで随時切り替え**（Phase 1〜2 の実装）
2. **初期設定（config）で決定し、通常操作では切り替えない**
3. **URL パラメータやサブドメインで切り替え**

## 決定

**初期設定で使用プロファイルを決定し、configに保持する。**  
設定画面でのみ変更可能とし、ヘッダーのプロファイル切り替えボタンは削除する。

## 理由

- 仕事中に個人ポータルを開く、またはその逆のシーンは実際にはほぼ発生しない
- 切り替えボタンは誤操作のリスクになる（仕事データと個人データの混在）
- 設定を「どちらのリポジトリを使うか」に絞ることで、UIがシンプルになる
- PAT が異なるため、切り替えは実質的にアカウント切り替えに近い操作

## 実装方針

- `portal_active_profile` の初期値はセットアップ時に設定画面で1回だけ設定
- ヘッダーには現在のプロファイル名を**表示のみ**（ボタンなし）
- 設定画面で「使用プロファイル」を変更した場合はページをリロードして反映

## トレードオフ・リスク

- 1台のデバイスで仕事/個人を切り替えたい場合は設定画面を経由する必要がある
- ブラウザプロファイル（Chrome プロファイル等）で切り替える運用を推奨する

## 将来の再評価タイミング

「仕事中に個人タスクも見たい」という具体的なユースケースが発生した場合。
