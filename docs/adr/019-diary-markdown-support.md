# ADR-019: 日記アプリでのマークダウン(Markdown)記述・表示のサポート

日付: 2026-04-21
状態: 提案（検討中）

## 背景

現在の日記作成機能はプレーンテキストでの入力が想定されていますが、日々の記録を残すうえで、見出し、箇条書きリスト、強調（太字）、リンク、コードブロックなどを利用し、よりリッチで表現力のある記述をしたいという要望があります。
マークダウン (Markdown) 形式での記述と同形式での表示レンダリングをサポートすることで、可読性や表現力を向上させたいと考えています。

## 課題と検討事項

マークダウン対応にあたり、以下のポイントについて検討・決定する必要があります。

### 1. 表示用のレンダリングライブラリの選定
入力されたマークダウン文字列をReact上で安全かつ美しく表示するためのパーサーを選定します。
*   **候補:** `react-markdown` や `marked` など。
*   **考慮点:** サニタイズ（XSS対策として `rehype-sanitize` のようなプラグインを噛ませるか）と、GitHubスタイルのマークダウン (`remark-gfm`) への対応。

### 2. エディタ（入力画面）の体験
単なる `<textarea>` を使い、Markdownのプレビュー画面（「書く」タブと「プレビュー」タブの切り替え）を用意するか、WYSIWYGライクなリッチエディタを導入するかの選択になります。
*   **プレーンな実装:** 既存のテキストエリアをそのまま使い、隣またはタブ切り替えで `react-markdown` のプレビューを表示させる。
*   **エディタライブラリの導入:** `EasyMDE` やその他のReact対応マークダウンエディタライブラリを使用して、入力支援（ショートカットで太字にする等）を強化する。

### 3. データ保存形式への影響
*   現状Vaultなどにデータを保存している場合、文字列として保存する分にはこれまでのプレーンテキストとの互換性は保たれます。
*   必要に応じてファイルの拡張子を `.md` と明確にしたり、JSONでの保存の場合はエスケープ処理に気をつける必要があります。

## 決定事項（仮）

**(現在検討中・未定)**
実装の第一歩としては、最も導入コストが低い **`textarea` での記述 + `react-markdown` を用いたプレビュー表示（ビュー画面でのレンダリング対応）** から進めることを推奨します。エディタの高度化については、基礎を実装した後の追加検討事項とします。
ユーザー回答：推奨に従う、プレーンな実装で良い
