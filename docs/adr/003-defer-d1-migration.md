# ADR-003: Cloudflare D1 移行を音声機能追加まで延期

日付: 2026-04-07  
状態: 採用

## 背景

将来的に音声入力・全文検索・集計機能を追加する予定がある。これらの機能はMarkdown + GitHub API では実現が難しく、SQLiteベースのDBが適している。Cloudflare D1（エッジSQLite）が候補として上がっている。

## 検討した選択肢

1. **今すぐ D1 に移行**（アーキテクチャを先行して整備）
2. **音声機能追加タイミングで移行**（今は GitHub API 継続）
3. **D1 は採用せず GitHub API を継続**（集計はクライアント側で）

## 決定

**今フェーズは GitHub API を継続し、音声機能追加タイミングで D1 移行を検討する。**

ただし、移行を容易にするために `storage/interface.ts` に StorageAdapter インターフェースを定義しておく。現実装は `storage/github.ts`、将来実装は `storage/d1.ts` として差し替え可能にする。

## 理由

- 現時点では D1 を使う機能（全文検索・集計）が存在しない
- Cloudflare Workers の設定コストが今フェーズには過大
- StorageAdapter パターンで実装しておけば、移行時にインターフェース層より上は変更不要
- YAGNI（You Aren't Gonna Need It）の原則に従い、実際に必要になるまで実装しない

## トレードオフ・リスク

- GitHub API の全文検索・集計能力の限界（ファイル数が増えると遅くなる）
- 後からDB移行する際にデータマイグレーションが必要

## 将来の再評価タイミング

以下のいずれかが発生した場合:
- 音声入力機能を追加する
- 日記ファイルが500件を超えて検索が遅くなる
- 月次集計など集計クエリが必要になる
